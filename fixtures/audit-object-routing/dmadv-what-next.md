# Fixture: DMADV What Next / Features / Roadmap

Input shape:

```text
Using /implementaudit, what next should this repo build after v0.3.0.0?
```

Expected route:

- DMADV direction/design
- future capability planning inside the `tdqyq-audit-object`
- separated from DMAIC defect findings

Required behavior:

- ground direction candidates in live repo evidence;
- define the direction question and owner/user served;
- measure current constraints and evidence gaps;
- analyze bounded alternatives;
- design the selected candidate as spike / phase / defer / reject or owner
  decision;
- include acceptance, risk, verification, rollback, and owner/source.

Forbidden behavior:

- do not return generic roadmap prose;
- do not mix defects and direction candidates without route labels;
- Do not advertise `/implementaudit next`, `/implementaudit features`, or `/implementaudit roadmap`.

Evidence required:

- live source/doc/checker evidence behind each candidate;
- route label: DMADV direction/design;
- outcome label: spike / phase / defer / reject / owner decision;
- verification and rollback notes.
