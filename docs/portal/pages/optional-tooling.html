Evidence

Optional tooling

Optional orientation and custody tools: useful for maintainers, never required for normal runs.

You do not need these tools for a normal run.

Markdown work records, ledgers, final reports, and run folders when phased are the durable fallback.
Graphify and ActiveGraph help maintainers inspect terrain and event custody when they are present and authorized.
They do not replace real files, before/after checks, or the final audit ledger.

Graphify

Graphify can help orient around repo terrain before mutation by showing structure and relationships.
In this repo, generated terrain may live under graphify-out/ or a graph JSON file, but it remains support evidence unless the run explicitly authorizes and records its use.

Repo convention: terrain lives at graphify-out/graph.json; nodes have id and label; links have source and target.
If tracked files are newer than the terrain, treat it as stale and let live files win.

ActiveGraph

ActiveGraph can preserve custody events and Capability Ledger-style traces when available.
Event stores help explain how a run moved through gates, but broad capability claims still need to be derived from recorded gates and evidence.

ActiveGraph precision

ActiveGraph is optional custody infrastructure, not correctness proof.

  • Do not turn one small run into a broad capability or resume-style claim.
  • Custom event names such as repo.patch.applied describe this repo's evidence model; they are not upstream proof by themselves.
  • activegraph.toml is a configuration clue, not authorization to write events.
  • Live custody and historical backfill must be labeled differently.

Authorization gates

Tool install, indexing, event-store setup, and export are separate owner-authorized actions.

  • Do not install Graphify without authorization.
  • Do not index a repo without authorization.
  • Do not configure ActiveGraph without authorization.
  • Do not export sidecar data without authorization.

Self-maintenance sidecars

For IMPLEMENTAUDIT maintaining itself, Graphify and ActiveGraph are canonical and expected dogfood sidecar inputs when present and authorized.
For consuming repos, they remain optional.
Live files, AGENTS.md, run-folder state, before/after evidence (Smoke A/B), and final audit closure still outrank sidecars.

Sidecar conventions

Graphify terrain

Concrete convention
graphify-out/graph.json; nodes use id + label, links use source + target; excluded from package and tracked source.
Proof boundary
Orientation only; stale when tracked files are newer; not proof of correctness.

ActiveGraph custody

Concrete convention
One <run-root>/custody.db or <run-root>/custody-trace.jsonl store per run folder when configured and authorized.
Proof boundary
Event custody only; not proof a change is correct.

Andon custody events

Concrete convention
andon.probe.recorded, andon.escalated, and andon.handoff.recorded can mirror failure handling.
Proof boundary
Preserves failure chain; does not replace the Markdown Andon log.

Run-level sidecar status

Concrete convention
sidecars.md records availability, authorization, output summaries, and fallback.
Proof boundary
Status evidence; does not authorize setup, export, or indexing.

Historical backfill

When older evidence is loaded into a sidecar, label it as historical backfill rather than pretending it was captured live during the current run.
This keeps custody evidence honest.

A historical backfill record should name:

  • custody_mode: historical_backfill
  • source
  • backfilled_at
  • original_event_time
  • evidence_boundary.

Capability Ledger (narrow evidence record)

A Capability Ledger is a narrow record of what this run evidenced.

  • It can be backed by Markdown or authorized ActiveGraph events.
  • When ActiveGraph is configured and authorized, entries can be event-backed.
  • The Markdown ledger remains first-class when ActiveGraph is absent, stale, or out of scope.
  • No ledger entry should turn a small fix into a general competence claim or infer success from chat memory.

When ActiveGraph is configured and authorized, a narrow Capability Ledger entry may include:

  • repo
  • run id
  • owner/source
  • quality route
  • quality-route and reflection notes
  • whether Graphify terrain was used
  • ActiveGraph event ids
  • checks run
  • final status
  • remaining risk.

Fallback

Missing optional tooling does not block a run.
Markdown work records, ledgers, final reports, and run folders when phased remain the durable fallback.

Source surfaces

README.md optional tooling, skills/templates/sidecars.md, and skills/references/lean-operating-discipline.md.

Absent-safe rule

If Graphify or ActiveGraph is missing, stale, or unauthorized, continue with live file inspection and Markdown ledgers.
Missing optional tooling is not a blocker for ordinary consuming repos, and dogfood sidecars still do not prove correctness by themselves.

When optional tools help

Terrain maps help owner/source discovery.
Custody events help trace proof events.
Neither replaces before/after checks, final audit, or source evidence.
