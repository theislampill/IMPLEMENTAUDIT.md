Audience

For auditors and maintainers

Use this path when reviewing whether a run, docs claim, package boundary, or release note is truthful to evidence.

Review order

  1. Read the request and bound work record; for phased work, inspect the run folder, STATE.md, roadmap, phase specs, and failure rows.
  2. Check STATE.md, roadmap, phase specs, and failure rows.
  3. Compare before/after evidence and repo-state output.
  4. Verify marker order: AUDIT_COMPLETE before IMPLEMENTAUDIT_RUN_COMPLETE.
  5. At release gates, verify scripts/check-forbidden-terms.sh ran with forbidden identity terms supplied at runtime and recorded PASS/FAIL without writing those terms into tracked files.
  6. Reject overclaims about install proof, host load, publication, provenance, sidecar proof, or package contents.

Sidecar boundary

Graphify and ActiveGraph are canonical and expected dogfood sidecar inputs for this repo when present and authorized, but they are optional for consuming repos.
Graphify is orientation evidence.
ActiveGraph custody is not correctness proof.

Plugin cache warning

Claude plugin cache refresh depends on a version-field change.
Shipped code changes need a version bump before claiming update behavior; that warning is not a release or install claim.
