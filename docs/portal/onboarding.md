## What is IMPLEMENTAUDIT?

IMPLEMENTAUDIT is an audit-governed implementation skill for repo changes.
It inspects the live owner/source, patches bounded surfaces, runs Smoke A/B
before and after changes, closes every ledger item terminally, and stops
honestly when blocked — emitting an explicit audited handoff instead of fake
completion.

Every finding closes. No orphan items. No unsafe actions. No proof claim without
evidence.

## When should I use it?

Use IMPLEMENTAUDIT for:

- Closing audit findings, handoffs, checklists, or reviews
- Implementing bounded implementation plans or gap-closure specs
- Repair of existing repo defects, regressions, or broken checkers
- Package-boundary repairs and generated-artifact freshness checks
- Stale docs, CI/checker work, and safety-sensitive repo changes
- Any repo work where you need auditable, owner/source-grounded, reversible
  changes with smoke-checked evidence and explicit authorization boundaries

IMPLEMENTAUDIT also accepts natural-language repo-build intent through governed
casual-build intake: describe your goal, and the skill synthesizes a bounded
audit object before routing to the appropriate work mode.

## When should I not use it?

Do not use IMPLEMENTAUDIT when you want ungated, unbounded automation that
bypasses owner/source discovery, acceptance criteria, rollback/defer planning,
Smoke A/B, the final audit, or authorization boundaries.

Casual repo-build intent is valid only when IMPLEMENTAUDIT can first synthesize
a bounded audit object. If you cannot define owner/source, acceptance criteria,
and rollback path, clarify those first.

## Three invocation modes

**Direct governance** — bind the audit object from a supplied audit, handoff,
checklist, review, or implementation plan:

    /implementaudit close the findings in AUDIT.md

**Embedded governance** — inherit the audit object from an outer /goal runner:

    /goal using /implementaudit, close the findings in AUDIT.md

**Goal synthesis** — synthesize a bounded audit object from an idea or gap:

    /implementaudit audit this repo state and give me the next best implementation goal

**Governed casual-build intake** — synthesize an audit object from natural-language
intent before routing to the appropriate work mode:

    /implementaudit make the docs portal generated by CI and prove it is fresh

## Compared with a generic staged-goal runner

Generic staged-goal tools often optimize for low-friction autonomous build
execution, including per-phase memory snapshots, personal/cross-project
continuity, optional terminal polish/harden phases, and casual invocation from
a plain description.

IMPLEMENTAUDIT now absorbs the useful parts through governed casual-build intake,
bounded continuity preload, per-phase continuity writeback, and optional
polish/harden phases. The difference is that IMPLEMENTAUDIT does not let those
conveniences bypass owner/source, Smoke A/B, rollback, authorization boundaries,
Graphify/ActiveGraph evidence boundaries, the final audit, or audited handoff.

No all-domain obsolescence claim is made. Each tool has its own scope and users.

## What Graphify and ActiveGraph do

**Graphify** provides optional terrain and orientation evidence. It can help
identify owner/source candidates, dependency paths, and stale assumptions. It is
not proof. Graphify absence is not an error.

**ActiveGraph** provides optional chain-of-custody evidence for audit gate
passages. It is not correctness proof. ActiveGraph absence is not an error.

Both sidecars are optional for users running IMPLEMENTAUDIT on their own repos.
For IMPLEMENTAUDIT self-maintenance runs, both are canonical.

## What completion means

Completion means:

- Owner/source changes made and verified
- Generated outputs regenerated from their source generators
- Smoke A (before mutation) and Smoke B (after mutation) recorded
- Every ledger item terminally closed as done, changed, blocked, deferred, or unverified
- AUDIT_COMPLETE printed before IMPLEMENTAUDIT_RUN_COMPLETE

It does not mean "some deliverable exists." Blocked work ends in an explicit
AUDIT_HANDOFF, not fake completion.
