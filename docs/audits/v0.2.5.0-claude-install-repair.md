## Diagnosis

**Primary root cause:** The `.skill` archive had the wrong root shape. Skill content
was nested under `skills/SKILL.md` in the archive, but Claude Desktop's import
contract requires `SKILL.md` at archive root. The Codex installer explicitly reads
from `skills/` so it accepted the archive; Claude Desktop rejected it with
"malformed at root."

**Secondary root cause:** No live Claude Desktop import was run in the release gate.
The gate ran `tests/release-asset-install.test.sh` (Codex-only). The README Claude
section said only "use the host's current plugin instructions" without any concrete
command. No `install-claude-from-release.sh` existed; no Claude archive smoke test.

## 5 Whys

1. Why can't users install v0.2.5.0 in Claude Desktop? → Archive root is `skills/SKILL.md`; Claude Desktop import requires `SKILL.md` at root.
2. Why was the archive root wrong? → `build-release-asset.sh` iterated `skills/` and kept the `skills/` prefix in archive paths; no Claude import test caught it.
3. Why was there no Claude import test? → No Claude-specific install/import test existed; Codex test accepted the nested-root archive.
4. Why did the Codex test not catch the root shape error? → Codex installer explicitly extracts from `skills/` subdirectory; it works with the wrong shape.
5. **Countermeasure:** Fix archive builder to put content at root; add regression guard (`skills/SKILL.md` in archive = hard failure); add Claude archive smoke; enforce in AGENTS.md anti-repeat rule.

## Hansei

Gap: Claude install coverage was absent from release gates since v0.2.3.0.
Cause: the first Codex install script (`install-codex-from-release.sh`) was never
accompanied by a corresponding Claude installer.
Countermeasure: the repair adds both the script and the test, plus an anti-repeat
rule in AGENTS.md requiring both Codex and Claude archive smoke before future release gates.
Follow-up evidence: the new `tests/release-asset-install-claude.test.sh` must pass
in CI (added to `.github/workflows/validate.yml`) and in future release gates.
