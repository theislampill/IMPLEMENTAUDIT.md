## Diagnosis

**Root cause:** Codex-only install assumptions. The release gate ran
`tests/release-asset-install.test.sh` which installs to a temporary Codex-style
`~/.codex/skills/` directory. There was no equivalent for Claude Desktop. The
README Claude section said only "use the host's current plugin instructions"
without any concrete command or path. No `install-claude-from-release.sh`
script existed. No `tests/release-asset-install-claude.test.sh` existed.

## 5 Whys

1. Why can't users install v0.2.5.0 in Claude Desktop? → No documented path with working commands.
2. Why was there no documented path? → Release gates only ran Codex install smoke.
3. Why was Claude omitted? → No Claude-specific install test existed to run.
4. Why did no Claude test exist? → No `install-claude-from-release.sh` script existed to test against.
5. **Countermeasure:** Add Claude install script + archive smoke test + enforce both in release gates via AGENTS.md anti-repeat rule.

## Hansei

Gap: Claude install coverage was absent from release gates since v0.2.3.0.
Cause: the first Codex install script (`install-codex-from-release.sh`) was never
accompanied by a corresponding Claude installer.
Countermeasure: the repair adds both the script and the test, plus an anti-repeat
rule in AGENTS.md requiring both Codex and Claude archive smoke before future release gates.
Follow-up evidence: the new `tests/release-asset-install-claude.test.sh` must pass
in CI (added to `.github/workflows/validate.yml`) and in future release gates.
