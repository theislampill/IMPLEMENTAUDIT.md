## Finding

The shipped skill package contradicted its own Lean/Jidoka identity:

- `skills/SKILL.md` (canonical identity) was correct: execute until terminal
  audit closure or an explicit audited handoff; no artificial phase cap;
  completion valid only via `AUDIT_COMPLETE`.
- `skills/references/transcript-contract.md` (ships in the package payload)
  defined a terminal three-strike sequence: FAILURE_PROBE → FAILURE_ESCALATE
  → FAILURE_HANDOFF, where "Strike 3" set STATE.md to BLOCKED, stopped the
  run, and declared "No subsequent phases execute." Behavior-affecting, not
  merely repo docs.
- `skills/templates/PROTOCOL.md` shipped the same strike ladder plus
  "Maximum 3 rounds" / "up to 3 rounds" caps on final-audit audit-fix rounds.
- `docs/diagrams/execution-spine.mmd` and the generated README block
  advertised "3-strike failure recovery" and "up to 3 rounds."
- Origin: the v0.2.6.0 hardening round intentionally adapted an external
  comparator's "exact 3-strike failure recovery" and "3-round audit-fix loop"
  (see `docs/audits/v0.2.6.0-operational-parity-hardening.md`). The anti-skip
  ordering rule was correct; the terminal try/round caps overcorrected into a
  Jidoka contradiction (handoff driven by a counter, not a blocking condition).

## Countermeasure

Jidoka loop restored as the failure contract:

```text
abnormality -> stop -> understand why -> countermeasure -> rerun evidence
```

Markers renamed and re-scoped from test-failure-only to any abnormality:

| Marker | Trigger | Must record |
|---|---|---|
| `ANDON_PROBE` | first abnormality: failed criterion, regression, hung/substituted command, unclear owner/source, generated-artifact mismatch, stale sidecar, policy conflict, impossible criterion, evidence mismatch | abnormality; failing criterion/command/artifact; owner/source; containment decision; proportional 5 Whys; Hansei; countermeasure selected; rerun evidence required. A fix must follow from the probe, not the visible symptom. |
| `ANDON_ESCALATE` | first countermeasure fails, same-class abnormality recurs, root cause unclear, fix would expand scope, owner/source disputed | prior probe history; why the first countermeasure failed; revised/deeper 5 Whys; chosen path (split, reframe, rollback, owner decision, bounded fix-spec); rerun evidence |
| `ANDON_HANDOFF` | only when closure is blocked by owner decision, unsafe scope, missing authorization, external dependency, irreproducibility, missing required tool/access, or no bounded countermeasure remains | full probe/escalation history; blocking condition; remaining blocker; smallest next owner action. It is not "third try failed." |

No arbitrary three-try or three-round cap anywhere. Final-audit audit-fix
rounds loop until closure or audited handoff. Ordering is preserved
(probe before escalate before handoff; no skipping to handoff on the first
abnormality), which keeps the valid half of the v0.2.5.0 anti-skip rule.
