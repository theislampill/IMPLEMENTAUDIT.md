# v0.3.0.0 Runtime Terminology Force-multiplier Audit

## Verdict

Status: closed for local source/package/docs verification on 2026-06-13. This
audit accepted the KISS Andon and changed the implementation target from a
broad runtime terminology reference to a thin terminology-integration contract
plus owner-reference hooks.

## KISS Andon

Andon class: evidence-mismatch.

Blocker: the first implementation draft created
`skills/references/runtime-terminology.md`, a 195-line second glossary. Gemba
showed the repo already had owning references for Lean discipline, routing,
phase design, plan lifecycle, security pressure, and audit category routing.

Countermeasure: delete the second glossary, preserve only the missing
cross-reference/precedence contract, and merge term-specific behavior into the
owning references.

## Why terminology-integration.md remains justified after KISS reduction

`skills/references/terminology-integration.md` remains justified because no
single existing reference owns cross-reference precedence across external term
families, native runtime parents, orphan-term rejection, and glossary-theater
rejection.

Evidence:

- Deleted file: `skills/references/runtime-terminology.md`, 195 lines.
- Replacement file: `skills/references/terminology-integration.md`, 71 lines,
  63.6% smaller.
- Forbidden second-glossary shapes are absent: no `## Term Entries`, no
  `- Term:`, no `Plain behavior`, no `Formal label`, no per-term subsection
  definitions.
- The replacement file states precedence, ownership, orphan-term rejection,
  glossary-theater rejection, and compact integrated flow.
- A3, Poka-yoke, Standard Work, DMAIC, DMADV, Jidoka, Andon, Plan Closure, and
  sidecar boundaries are not redefined there; the file points to
  `lean-operating-discipline.md`, `routing.md`, Plan Closure, Smoke A/B, and
  final audit as the authorities.
- Term-specific behavior is owned elsewhere:
  - `routing.md`: VOC/CTQ/SIPOC and Strangler/ACL mixed replacement.
  - `planning-depth.md` / `phase-design.md`: FMEA-lite risk shaping and
    orphan-term phase rule.
  - `lean-operating-discipline.md` / `plan-lifecycle.md`: Control Plan,
    Poka-yoke, and Standard Work sustainment hooks.
  - `audit-category-matrix.md`: STRIDE/trust-boundary/repo-content-as-data
    security pressure.
  - `scripts/check-terminology-integration.sh` and
    `fixtures/terminology-integration/`: SOLID/GRASP generic-advice rejection.

KISS acceptance decision: keep the thin file as a cross-reference/precedence
contract only. If it grows into definitions, term entries, duplicate Lean/routing
authority, or a separate terminology lane, the checker must fail and the content
must be merged into existing references or deleted.
