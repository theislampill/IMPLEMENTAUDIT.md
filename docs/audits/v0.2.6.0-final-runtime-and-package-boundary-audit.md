### Compression verification

As of v0.2.6.0, the build script explicitly sets `info.compress_type = zipfile.ZIP_DEFLATED`
on each `ZipInfo` object before calling `zf.writestr()`. Without this, the
`ZipInfo.compress_type` attribute (which defaults to `ZIP_STORED = 0`) silently
overrides the `ZipFile`-level `compression=ZIP_DEFLATED` default.

`tests/release-asset.test.sh` now verifies:
1. Every non-empty entry has `compress_type != ZIP_STORED` (per-entry check on `ZipInfo.compress_type`).
2. Total asset size is ≤ 120,000 bytes (2× headroom over the current ~60 KB baseline, well below the ~155 KB uncompressed artifact).

Both checks are mandatory on every `release-asset.test.sh` run.
