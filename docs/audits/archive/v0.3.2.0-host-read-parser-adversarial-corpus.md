## Contract vocabulary

- `content-read`: the normalized host evidence proves that the exact target file
  content was successfully read.
- `not-content-read`: the evidence proves the action did not read the exact target
  content (for example, it printed or listed the filename).
- `fail-closed`: the event is malformed, ambiguous, unsupported, or lacks enough
  provenance to establish a content read. It must not manufacture positive
  evidence.

The production API needs the three states above. A Boolean reader predicate
cannot preserve the distinction.

Test identities are literal strings:

```text
S=.IMPLEMENTAUDIT/runs/run-1/STATE.md
R=.IMPLEMENTAUDIT/runs/run-1/ROADMAP.md
W=.IMPLEMENTAUDIT/runs/run-1/capsule.json
```

Unless a row supplies a different envelope, its command is the first event and
the native file-change is the second event:

```json
{"type":"item.completed","item":{"type":"command_execution","exit_code":0,"command":"<COMMAND>","aggregated_output":"<OUTPUT>"}}
{"type":"item.completed","item":{"type":"file_change","changes":[{"path":".IMPLEMENTAUDIT/runs/run-1/capsule.json","kind":"add"}]}}
```

When both exact durable files are required, a PASS requires `S=1`, `R=1`, and
`W=2`. A partial or ambiguous read must keep the two-file gate false.

## Retained GREEN controls

- Direct, dialect-bound and executable-attested `cat`, `sed`, `head`, `tail`,
  `Get-Content`, cmd `type`, `rg`, and `grep` with exact file operands.
- The actual host-owned Codex `/bin/bash -lc` envelope with paired start and
  completion records.
- Listing-only `find -type`, `rg --files`, `ls`, `stat`, `git status`, and
  synthetic `printf` path output remain non-reads.
- Native successful Read/Read/Write ordering passes; write-before-read and
  split-around-write fail.
- Unclosed quotes and unsupported nested shells fail closed.
- File output redirections fail closed.
- POSIX quoted or escaped literal `>` data is not a redirection.
- Existing transparent wrappers such as `env -i`, `sudo -n`, `command cat`,
  `exec cat`, and `xargs -0 cat` retain their behavior.

## Explicit contract decisions, not automatic regressions

1. Normalize a host/shell identity before interpreting dialect-specific `type`,
   PowerShell backticks/stream merges, Windows path escaping, or case behavior.
2. Keep nested interpreters, aliases, functions, arbitrary programs, and dynamic
   call operators fail-closed without reviewed nested parsing or host access
   telemetry.
3. Keep variables, braces, globs, `cd &&` relative paths, symlinks, and other
   expansion-dependent identities fail-closed unless expanded argv or accessed
   paths are retained.
4. Compound/newline/group/background semantics require an explicit
   status-entailment contract.
5. Additional readers (`nl`, `nice`, `stdbuf`, `busybox`, PowerShell aliases,
   `Select-String`) are optional support, not automatic defects.
6. Anonymous directory-search output cannot prove which files were read.
   Structured filename output may be supported only with structural parsing and
   declared search-scope validation.
7. Resolve identity against the fixture repository and actual filesystem; never
   lowercase or suffix-match unconditionally.
8. A shell redirection to W without a native write event is incomplete evidence
   unless transparent shell writes are separately normalized.
9. Freeze the accepted Codex schema before deciding outer-status and numeric-zero
   compatibility.
