## 1. Executive verdict

Yes: `/improve` can be behaviorally subsumed by `/implementaudit`, if "subsumed" means every useful donor capability lands inside IMPLEMENTAUDIT's stronger audit-object lifecycle rather than becoming copied command identity. The donor's practical value is advisor-grade repo survey, category pressure, plan quality, branch scoping, dispatch/review semantics, reconciliation, and issue-publication ergonomics. The target already owns a stronger execution contract: `tdqyq-audit-object`, `ydqyq-audit-action`, owner/source routing, DMAIC/DMADV routing, Smoke A/B, Andon, final audit, run roots, authorization gates, sidecar boundaries, and package validation.

Intentionally not copied:

- The donor's read-only identity as the default behavior. IMPLEMENTAUDIT remains audit-governed implementation, not a generic advisor.
- The donor command surface as separate IMPLEMENTAUDIT modes. There must be no advertised `/implementaudit deep`, `/implementaudit security`, or `/implementaudit next`.
- The donor root `plans/` directory as canonical. IMPLEMENTAUDIT canonical planning substrate remains `.IMPLEMENTAUDIT/runs/<slug-id>/`.
- The donor max-two revision cap in `execute`. IMPLEMENTAUDIT must preserve Jidoka Andon escalation with no arbitrary try or revision cap.
- Donor `--issues` implementation in v0.3.0.0. It is audited and deferred behind a future explicit publication gate.

v0.3.0.0 is the right milestone because this is a behavior-level expansion from v0.2.9.0: it affects default audit pressure, route semantics, run-root templates, phase validation, child-agent/review loops, reconcile flows, docs, fixtures, and package validation. It is too broad for a patch release but does not require a new release/provenance lane in this planning run.

The plan preserves IMPLEMENTAUDIT's identity: audit-object lifecycle first, donor advisor ergonomics second. Donor capabilities become default pressure, DMADV direction routing, self-contained run-root plans, bounded dispatch/review semantics, and reconciliation rules inside the existing gates.
