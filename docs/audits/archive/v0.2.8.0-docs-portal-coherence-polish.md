## Scope

A reader-journey audit of the docs portal after the academic/content-model repair (commit f9d300c). The portal had the right broad structure but needed coherence, precision, and prose quality improvements. No architecture changes were in scope.

Three reader perspectives audited:
- **First-time user:** "What is this and when do I use it?"
- **Agent/operator:** "How do I run it correctly?"
- **Maintainer/auditor:** "What does the evidence prove and where is the canonical source?"

## Findings

### F1. Callout title rendering bug — P1 (generator defect)

**Symptom.** All callout titles rendered with a double period (`..`) at the end. Titles containing backtick code spans (e.g., `` `IMPLEMENTAUDIT_RUN_COMPLETE` ``) rendered with literal backticks instead of `<code>` elements.

**Root cause.** `render_callout()` in `build-docs-portal.py` called `esc()` instead of `inline_md()` on the title text, and unconditionally appended `.` to a title that already ended with `.`.

**Fix.** Generator: strip trailing period/space from title before appending; apply `inline_md()` (not bare `esc()`) so backtick code spans render correctly.

**Evidence type.** Static checker (generated HTML inspection).

---

### F2. "Does not mean" list — redundant "Not " prefix — P1 (prose defect)

**Symptom.** Each item in the `AUDIT_COMPLETE` "does not mean" list started with "Not ..." — redundant given the heading "AUDIT_COMPLETE does not mean:". Items read awkwardly as double-negative constructions.

**Fix.** Reformatted as `Term — explanation` pattern. Changed last item from `Not "the agent felt done."` to `A felt sense of completion — evidence determines terminal state, not the agent's confidence.`

---

### F3. "absorbs" heading — product-y language — P1 (prose defect)

**Symptom.** Comparison section subheading "What IMPLEMENTAUDIT absorbs from staged-goal runners" used an informal product-y verb ("absorbs") inconsistent with the method-document register.

**Fix.** Changed to "Behaviors shared with staged-goal runners."

---

### F4. Section order — Default Behavior before Usage Examples — P2 (reader-journey defect)

**Symptom.** The Method group ordered Default Behavior (abstract) before Usage Examples (concrete). Operator readers wanting examples had to skip past the behavior spec first.

**Fix.** Swapped: Usage Examples now precedes Default Behavior in both the sidebar SIDEBAR_GROUPS and the document section order. Test 25 (nav-section anchor order matches document order) verified.

---

### F5. Terminology table missing continuity preload/writeback — P2 (completeness defect)

**Symptom.** "Continuity preload" and "continuity writeback" were referenced throughout the Concepts and Comparison sections but not defined in the Terminology table.

**Fix.** Added both terms with definitions including the five-source priority order (preload) and five-option CONTINUITY_DECISION mechanism (writeback).

---

### F6. Mental Model ydqyq-audit-action example — confusingly terse — P2 (prose defect)

**Symptom.** "Implementation is an `ydqyq-audit-action`. A passing CI check is not." — the contrast was correct but ambiguous (CI checks ARE evidence; the distinction is whether the operation mutates the audit object).

**Fix.** Changed to "Patching owner/source is an `ydqyq-audit-action`; a passing CI check is evidence but does not itself mutate the audit object."

---

### F7. Evidence and Audit Trail — missing coherence-polish audit reference — P2 (completeness)

**Symptom.** The Evidence section didn't reference this audit once written.

**Fix.** Added reference to `v0.2.8.0-docs-portal-coherence-polish.md`. Also updated the `docs/diagrams/` line to name all three Mermaid files (execution spine, invocation modes, tooling architecture).

---

### F8. Audit Status "Comparator note" — internal meta-language — P2 (tone defect)

**Symptom.** "Comparator note: No all-domain obsolescence claim is made." was defensive internal meta-language that reads oddly in a public method-document status table.

**Fix.** Replaced with "Methodology scope: The comparison table covers methodology dimensions only. No all-domain obsolescence claim is made."
