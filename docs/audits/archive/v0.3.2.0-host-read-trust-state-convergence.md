## State family B — global typed action intervals

Every recognized or safe-other action reserves its ID globally and belongs to
exactly one bound host session/turn lineage:

```text
UNSEEN
→ PENDING(invocation ordinal, action identity, full payload)
→ UPDATED*(schema-valid intermediate updates for declared safe-other lifecycles)
→ COMPLETED(success/failure, completion ordinal)
or INVALID / INCOMPLETE

UNSEEN
→ TERMINAL_SAFE_MESSAGE(schema-bound completion-only message)
```

`TERMINAL_SAFE_MESSAGE` is the only legal completion-only Codex transition. It
is restricted to schema-valid `agent_message` items, reserves the ID globally,
and never invents an action invocation. A later reuse of that ID is INVALID.
`UPDATED*` is restricted to a schema-valid `todo_list` safe-other interval:
`PENDING(todo_list) → UPDATED* → COMPLETED`. Command, read, and write
completions still require their matching invocation.

## State family D — finite shell/reader semantics

Support contracts by attested dialect. Do not lowercase POSIX executable
identity or strip `.exe` outside a Windows profile. Attestation-changing
wrappers (`env -i`, PATH removal, sudo user transition, LD_PRELOAD) fail closed
unless the post-transition resolution is separately verified.

Process access is separate from the #35 product property. The frozen semantic
decision is: **#35 requires target content delivered to the model and bound to
the immutable pre-mission file before mutation.** Process-level access remains
a separate host fact. Native Read results and supported shell output must match
the bound preimage. Redirected, suppressed, transformed, partial,
un-attributable, or configuration-only reads fail closed for the product
property.

Claude's schema-bound native `Read` line-number framing is a narrow transport
exception, not a general transformed-output allowance. It qualifies only when
the structured `type: text` file content equals the immutable preimage, the
typed line metadata proves a complete `startLine: 1` slice, and removing the
strict sequential line prefixes losslessly reconstructs those same bytes.
Missing, reordered, partial, or contradictory visible lines fail closed.

Consequently `cat S >/dev/null`, pattern/config reads such as `grep -f S R`,
no-match searches, and partial `head` do not establish the full live-state
property even if the process opened S. A narrowly supported single-stage
`sed -n '1,320p' S` can establish the property only when output matches the
complete bound preimage; truncated or mismatched controls fail closed.

Scoped search output requires a specifically parsed filename-bearing format
and content-to-preimage binding. Suppressed, relabeled, replaced,
JSON/vimgrep/heading/null/custom separator, filename-only, and count-only output
remain fail-closed until explicitly supported.
