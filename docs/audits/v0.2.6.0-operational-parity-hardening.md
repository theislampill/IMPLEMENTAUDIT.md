### G3 — Copied run protocol too thin

**Status:** ADAPTED (Phases 6–8 of this run)

**Finding:** `skills/templates/PROTOCOL.md` in v0.2.5.0 describes the pattern at a high level but does not spell out:
- The concrete per-phase loop (read STATE.md → find current phase → read phase spec → print IMPLEMENTAUDIT_PHASE_START → do work → run mandatory commands → surface evidence → cleanliness check → print IMPLEMENTAUDIT_PHASE_VERIFY → continuity decision → print AGENTS_UPDATE_DECISION → print CONTINUITY_DECISION → print IMPLEMENTAUDIT_PHASE_DONE → update STATE.md → pause for interruption → continue)
- The exact failure recovery sequence (FAILURE_PROBE → retry → FAILURE_ESCALATE → fix spec → FAILURE_HANDOFF)
- The final audit protocol (AUDIT_START, re-read ROADMAP, phase completeness, re-run commands, spot-check criteria, deliverable check, AUDIT_VERIFY, AUDIT_GAPS, AUDIT_COMPLETE with coverage math, AUDIT_HANDOFF)
- Mid-run interruption handling
- STATE.md update steps
- Prohibition on IMPLEMENTAUDIT_RUN_COMPLETE before AUDIT_COMPLETE

**Decision:** Adapt. Expand PROTOCOL.md into a full executable loop with all steps. Keep IMPLEMENTAUDIT marker names and AGENTS_UPDATE_DECISION/CONTINUITY_DECISION structure.

**Owner/source:** `skills/templates/PROTOCOL.md`
**Implemented in:** Phases 6, 7, 8

### G12 — External identity hygiene not CI-owned by design

**Status:** INTENTIONALLY REJECTED with documentation (Phase 13 of this run)

**Finding:** v0.2.5.0 decided not to ship a source-owned external identity checker (checking for the external comparator's proper name in tracked files). The checker would need to embed the forbidden string to search for it, which risks checked-in identity. v0.2.5.0 left this as a manual release-gate audit step.

**Decision for v0.2.6.0:** Add a generic source-owned checker that accepts a caller-supplied forbidden-terms list at runtime (not embedded in source). Wire it into release-gate instructions in AGENTS.md. The forbidden term is never stored in tracked source; it's passed as a runtime argument during the release audit. This is the preferred middle path: CI-owned mechanism without source contamination.

**Owner/source:** `scripts/check-host-claims.sh` or new `scripts/check-forbidden-terms.sh`, `AGENTS.md`
**Implemented in:** Phase 13
